# 進捗管理ファイル (docs/claude-progress/)

複数セッションに渡るような大きめのissueに着手する際、進捗をこのディレクトリに
記録する。フレッシュなコンテキストでも、このファイルとgit履歴（コミット・
ブランチ・PR）を見ればすぐに作業を引き継げる状態を目指す。

## 命名規則

- `issue-<番号>.md`（例: `issue-68.md`）

## 運用ルール

- 大きめのissue（複数のサブステップに分かれる、または1セッションで
  完結しないと判断したもの）に着手する際、このディレクトリに
  進捗ファイルを作成する
- 作業の区切り（ステップ完了時、セッション終了時など）に進捗ファイルを更新する
- issueがクローズされ、対応するPRがマージされたら、進捗ファイルは削除する
  - 今後の判断に役立つ知見（設計判断の理由など）が含まれる場合は、
    削除前に`docs/adr/`にADRとして昇華させる

## テンプレート

```markdown
# Issue #<番号>: <タイトル>

## 対応中のissue/ブランチ

- Issue: #<番号>
- ブランチ: `<ブランチ名>`

## 完了したステップ

- [x] ...

## 残作業

- [ ] ...

## 判明した懸念事項

- ...
```
